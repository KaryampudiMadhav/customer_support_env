# Detailed Requirements - Customer Support Environment

## 1. Real-World Task Simulation

### 1.1 Domain: Customer Support Operations

The environment simulates a **customer support agent** handling inbound support tickets. The agent must:
- Understand customer issues (refunds, returns, billing disputes, technical problems)
- Apply company policies correctly (return windows, refund amounts, escalation rules)
- Maintain professional, empathetic tone
- Avoid hallucinations (inventing policies, false information)
- Make decisions that balance customer satisfaction with business constraints

### 1.2 Realistic Scenarios

**Why this matters:**
- Companies spend billions annually on customer support training and staffing
- Poor support decisions cost money (unjustified refunds, customer churn, chargebacks)
- Support agents must follow policies while managing frustrated customers
- LLM-based support agents need evaluation before production deployment

**Use Cases:**
- Training RL agents to improve support response quality
- Evaluating LLM agents (GPT-4, Claude) on policy compliance
- A/B testing refund policies and escalation rules
- Cost-benefit analysis of automation vs. human review

## 3. Task Specifications & Graders

### 3.1 Task 1: Simple Refund Processing (Easy) - 25% Weight

#### Scenario Details
- **Ticket Type**: Refund request
- **Context**: Product returned unopened, within 30-day window
- **Policy**: 100% refund for unopened returns within 30 days
- **Customer State**: Neutral, first contact
- **Complexity**: Straightforward application of policy

#### Grading Logic

```python
def grade_task1_simple_refund(action: CustomerSupportAction, order_info: dict) -> float:
    """
    Grade Task 1: Simple Refund
    
    Criteria:
    - Correct refund amount (0.4 points)
    - Polite tone (0.3 points)
    - No hallucinations (0.2 points)
    - Policy compliance (0.1 points)
    """
    score = 0.0
    
    # 1. Correct Resolution (0.4 points)
    if action.action_type == "refund":
        expected_amount = order_info["amount"]
        if abs(action.amount - expected_amount) < 0.01:
            score += 0.4
        elif abs(action.amount - expected_amount) < expected_amount * 0.05:
            score += 0.2  # Partial credit (rounding error)
    
    # 2. Polite Tone (0.3 points)
    tone_score = llm_grade_tone(action.response, context="positive_resolution")
    # Returns 0.0-1.0; 1.0 = excellent, 0.0 = rude
    score += min(tone_score, 1.0) * 0.3
    
    # 3. No Hallucination (0.2 points)
    if not detect_hallucination(action.response, order_info):
        score += 0.2
    else:
        score -= 0.1  # Penalty per hallucination
    
    # 4. Policy Compliance (0.1 points)
    order_date = parse_date(order_info["date"])
    days_since_order = (datetime.now() - order_date).days
    if days_since_order <= 30:
        score += 0.1
    
    return max(0.0, min(1.0, score))
```

#### Test Cases
1. **Baseline**: Unopened item, day 15, neutral customer → Score: 1.0
2. **Boundary**: Unopened item, day 30, frustrated customer → Score: 0.85 (tone penalty)
3. **Failure**: Day 35 (outside window) → Score: 0.0
4. **Partial**: Wrong amount (90% refund) → Score: 0.6

### 3.2 Task 2: Complex Refund with Partial Eligibility (Medium) - 35% Weight

#### Scenario Details
- **Ticket Type**: Refund request
- **Context**: Opened/used product, claims defect, day 29 of 30-day window
- **Policy**: 
  - Defective items: Refundable
  - Used items: 20% restocking fee
  - Time-sensitive: Must note 30-day deadline
- **Customer State**: Frustrated, escalating tone
- **Complexity**: Multi-factor decision (defect verification, fee calculation, tone management)

#### Grading Logic

```python
def grade_task2_partial_refund(action: CustomerSupportAction, order_info: dict, conversation: list) -> float:
    """
    Grade Task 2: Complex Refund with Restocking Fee
    
    Criteria:
    - Correct resolution (0.35 points)
    - Polite & empathetic tone (0.35 points)
    - Clear policy explanation (0.2 points)
    - No hallucination (0.1 points)
    """
    score = 0.0
    
    # 1. Correct Resolution (0.35 points)
    order_amount = order_info["amount"]
    expected_refund = order_amount * 0.8  # 20% restocking fee
    
    if action.action_type == "partial_refund":
        if abs(action.amount - expected_refund) < 0.01:
            score += 0.35
        elif abs(action.amount - expected_refund) < expected_refund * 0.1:
            score += 0.175  # Partial credit
    
    # 2. Polite & Empathetic Tone (0.35 points)
    tone_score = llm_grade_tone(action.response, context="frustrated_customer")
    empathy_score = detect_empathy_keywords(action.response)  # 0.0-1.0
    combined_tone = (tone_score + empathy_score) / 2
    score += min(combined_tone, 1.0) * 0.35
    
    # 3. Policy Explanation (0.2 points)
    if explains_restocking_fee(action.reason):
        score += 0.1
    if explains_deadline(action.reason):
        score += 0.1
    
    # 4. No Hallucination (0.1 points)
    if not detect_hallucination(action.response, order_info):
        score += 0.1
    
    return max(0.0, min(1.0, score))
```

#### Grader Helpers

```python
def explains_restocking_fee(reason: str) -> bool:
    """Check if reason mentions restocking fee or similar concept"""
    keywords = ["restocking", "handling", "fee", "20%", "used"]

def explains_deadline(reason: str) -> bool:
```

### 3.3 Task 3: Multi-Issue Escalation Decision (Hard) - 40% Weight

#### Scenario Details
- **Ticket Type**: Multiple issues (defect + shipping delay + poor service)
- **Context**: Return window closed (day 35), product defective, shipping late, customer very frustrated
- **Policy**: 
  - No refunds after 30 days (escalation required for exceptions)
  - Shipping delays qualify for 10% store credit
  - Policy violations require manager approval
- **Customer State**: Angry, threatening chargeback/social media
- **Complexity**: Multi-factor reasoning, escalation judgment, risk assessment

#### Grading Logic

```python
def grade_task3_escalation(action: CustomerSupportAction, order_info: dict, 
                           customer_history: dict, conversation: list) -> float:
    """
    Grade Task 3: Multi-Issue Escalation Decision
    
    Criteria:
    - Correct decision (0.3 points)
    - Clear justification (0.25 points)
    - Appropriate tone (0.25 points)
    - No hallucination (0.1 points)
    - Escalation judgment (0.1 points)
    """
    score = 0.0
    
    # 1. Correct Decision (0.3 points)
    days_since_order = calculate_days_since_order(order_info["date"])
    
    if action.action_type == "escalate":
        # Correct: window closed, needs manager review
        score += 0.3
    elif action.action_type == "partial_refund":
        # Offer store credit instead (within authority)
        if order_info["amount"] * 0.1 <= action.amount <= order_info["amount"] * 0.2:
            score += 0.2  # Partial credit (reasonable alternative)
        else:
            score += 0.0  # Wrong amount
    else:
        score += 0.0  # Denial without alternative is poor
    
    # 2. Justification Quality (0.25 points)
    justification_score = llm_grade_explanation(
        text=action.reason,
        criteria=["explains_window", "acknowledges_issues", "provides_path_forward"]
    )
    score += justification_score * 0.25
    
    # 3. Appropriate Tone (0.25 points)
    # Must be empathetic and professional despite angry customer
    tone_score = llm_grade_tone(action.response, context="angry_customer")
    empathy_score = detect_validation_keywords(action.response)  # Validates concerns
    score += min((tone_score + empathy_score) / 2, 1.0) * 0.25
    
    # 4. No Hallucination (0.1 points)
    if not detect_hallucination(action.response, order_info):
        score += 0.1
    
    # 5. Escalation Judgment (0.1 points)
    if action.action_type == "escalate":
        if is_appropriate_escalation(action.reason):
            score += 0.1
        else:
            score -= 0.05  # Unnecessary escalation
    
    return max(0.0, min(1.0, score))

def is_appropriate_escalation(reason: str) -> bool:
    """Verify escalation is justified (mentions manager, policy exception, etc.)"""
    keywords = ["manager", "exception", "policy", "review", "authority"]
    return any(kw.lower() in reason.lower() for kw in keywords)
```

#### Test Cases
1. **Baseline**: Escalate with clear justification, empathetic tone → Score: 1.0
2. **Tone Failure**: Correct escalation but dismissive → Score: 0.75
3. **Decision Failure**: Refuses without escalation/alternative → Score: 0.2
4. **Hallucination**: Offers non-existent "VIP exception" → Score: 0.35

### 4.3 Reward Function

```python
def calculate_step_reward(
    action: CustomerSupportAction,
    task_score: float,
    step_count: int,
    has_hallucination: bool
) -> float:
    """
    reward = task_score - time_penalty - hallucination_penalty
    """
    reward = task_score
    reward -= step_count * 0.02  # Penalize long responses
    if has_hallucination:
        reward -= 0.3
    return max(-1.0, min(1.0, reward))
```

```python
def validate_action(action: CustomerSupportAction, order_info: dict) -> tuple[bool, str]:
    """
    Returns (is_valid, error_message)
    """
    if action.amount and action.amount > order_info["amount"]:
        return False, "Refund amount exceeds order total"
    if action.action_type not in VALID_ACTIONS:
        return False, f"Invalid action type: {action.action_type}"
    if len(action.response) < 10:
        return False, "Response too short (min 10 chars)"
    if len(action.response) > 2000:
        return False, "Response too long (max 2000 chars)"
    return True, ""
```
